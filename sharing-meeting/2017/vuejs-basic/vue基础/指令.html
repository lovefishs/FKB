v-if | v-else-if | v-else

使用场景:

根据表达式的值在DOM中生成或移除一个元素

example:

<template>
    <div class="hello">
        <h1 v-if="ok">{{ isok }}</h1>
        <h5 v-else="ok">{{ isnot }}</h5>            // <h5 v-else>{{ isnot }}</h5>  等号内容可省略
    </div>
</template>

<script>
 export default {
      name: 'hello',
      data () {
           return {
                isok: 'ok = true, 显示h1标签内容',
                isnot: 'ok = false, 显示h5标签内容',
                ok: false
           }
      }
 }
</script>

v-show

使用场景：

根据表达式的值在DOM中显示或隐藏元素

example:

<template>
    <div class="vShow">
        <h4 v-show="greeting">{{ isShow }}</h4>
    </div>
</template>

<script>
    export default {
        name: 'vShow',
        data () {
           return {
                isShow: 'greeting = true, 显示h4标签内容, greeting = false, h4标签增加display: none样式',
                greeting: false
           }
        }
    }
</script>

结果：

注： v-show 不支持 template 语法

example:

<template class="vShow" v-show="greeting">
    <h4>{{ isShow }}</h4>
</template>

<script>
    export default {
        name: 'vShow',
        data () {
           return {
                isShow: 'greeting = true, 显示h4标签内容, greeting = false, h4标签增加display: none样式',
                greeting: false
           }
        }
    }
</script>

结果：

  • v-if 真实的条件渲染，它会确保条件块在切换时合适地销毁与重建条件块内的时间监听器和子组件，存在局部编译
  • v-show —— 元素始终被编译并保留，只是简单地基于CSS切换

v-if 有更高的切换消耗， 而 v-show 有更高的初始渲染消耗 需要高频切换使用v-show ， 反之使用 v-if

v-for

使用场景：

基于数据重复渲染元素

example:

<template>
    <div class="vFor">
        <ul >
            <li v-for="item in items"> {{item.name}}</li>
        </ul>
        <hr />
        <ul>
            <li v-for="value in object"> {{value}}</li>
        </ul>
        <hr />
        <ul>
            <li v-for="(value, key) in object">{{key}} - {{value}}</li>
        </ul>
    </div>
</template>

<script>
    export default {
      name: 'vFor',
      data () {
        return {
            items: [
                {
                    id: 1,
                    name: 'li 01'
                },
                {
                    id: 2,
                    name: 'li 02'
                },
                {
                    id: 3,
                    name: 'li 03'
                }
            ],
            object: {
                first: 'DIDI',
                next: 'FE',
                last: '32'
            }
        }
      }
    }
</script>

v-bind

使用场景：

将一个或对个attribute，或者一个组件 prop 动态绑定到表达式

example:

<template>
    <div class="vBind">
        <a v-bind:href="baidu">百度</a>
        <p :class="[classA, { classB: isB, classC: isC }]">class 绑定</p>
        <p v-bind="{ id: somProp, 'otherAttr': otherProp }">绑定多属性</p>
        <p :prop="someThing">prop</p>
        <p :prop.sync="someThing">prop.sync</p>
        <p :prop.once="someThing">prop.once</p>
    </div>
</template>

<script>
    export default {
      name: 'vBind',
      data () {
           return {
                baidu: 'http://www.baidu.com',
                classA: 'A',
                isB: false,
                isC: true,
                somProp: 'idName',
                otherProp: 'prop',
                someThing: 'someThing'
           }
        }
    }
</script>

结果：

补充：

- .sync —— 双向绑定，只能用于 prop 绑定
- .once —— 单次绑定，只能用于 prop 绑定
- .camel —— 将绑定的特性名字转换回驼峰命名。只能用于普通HTML特性的绑定，通常用于绑定用驼峰命名的SVG特性，比如 viewBox
